Module: bground.api

Definition of API for (application programming interface) of BGROUND package.

  • The API can be employed as a simple UI within Spyder and/or Jupyter scripts.
  • The main purpose of the API - easy access to functions for bkgr subtraction.

Short example:

>>> # Semi-automated background subtraction with InteractivePlot method
>>> # (in Spyder, use %matplotlib qt before running this script
>>>
>>> # Import API of BGROUND package
>>> import bground.api as bkg
>>>
>>> # Define I/O files
>>> INPUT  = 'ed1_raw.txt'
>>> OUTPUT = 'ed2_bcorr.txt'
>>>
>>> # Define data, plot parameters and background subtraction method
>>> DATA = bkg.InputData(INPUT)
>>> PARS = bkg.OutputParams(OUTPUT,'Pixels','Intensity',xlim=200,ylim=180)
>>> BMET = bkg.InteractivePlot(DATA, PPAR, CLI=False)
>>>
>>> # Run the selected BMET = Background subtraction METhod 
>>> # (here: interactive plot = define bkgr manually + subtract automatically
>>> # (follow the instructions in stdout + newly opened window showing XY-data 
>>> BMET.run()

More examples for the individual bkgr subtraction methods:

  • Semi-automated/interactive bkgr subtraction: bground.api.InteractivePlot
  • Restore background from saved bkgr points: bground.api.RestoreFromPoints
  • Other, fully-automated methods - under development

 59    '''

105class InputData:
106    '''
107    Class defining {InputData} for {InteractivePlot}.
108    
109    * Input data can be: file, np.array, pd.DataFrame, ediff.io.Profile.
110    * The usage of InputData class is shown in the example above.
111    * The rest of the documentation => detailed comments in the source code.
112    '''
113    
114    def __init__(self, input_data, **kwargs):
115        # Initialization of InputData object.
116        # No docstring: just class description above + comments below.
117        
118        # Call read_input_data method with **kwargs:
119        # The method reads input data and returns:
120        #  - self.name = name of the data, filename or variable name
121        #  - self.data = numpy array with two rows ~ X-data, Y-data
122        # The input_data may be:
123        #  - text file containing several columns
124        #    (then **kwargs are passed to pd.read_csv
125        #  - numpy array
126        #    (user responsibility: array contains just two rows ~ X,Y
127        #  - pandas DataFrame
128        #    (user responsibility: DataFrame contains just two cols ~ X,Y)
129        #  - data saved in Profile object from ediff package
130        #    (program responsibility: df['pixel'] ~ X, df['Iraw'] ~ Y
131        self.name, self.data, self.profile = \
132            self.read_input_data(input_data, **kwargs)
133        
134        
135    def read_input_data(self, input_data, **kwargs): 
136        # Read input data with XY-data.
137        # No docstring: just class description above + comments below.
138        # * The optional arguments (comment, sep, usecols)
139        #   are passed to pandas.read_csv if the input_data ~ filename.
140    
141        if isinstance(input_data, (str, Path)):
142            # The input is a file = filename given be str or PathLike var.
143            # => read the file using pd.read_csv
144            # => user's responsibility: modify optional args as needed
145            #    the optional arguments are: comment, sep, usecols
146            #    these arguments are passed to pd.read_csv
147            # (a) set name = filename
148            name = str(input_data)
149            # (b) get **kwargs for pd.read_csv OR set reasonable defaults
150            # ... comments may be present
151            my_comment  = kwargs.get('comment') or '#'
152            # ... header should be None if there are no column headers!
153            my_header   = kwargs.get('header') or 'infer'
154            # ... skiprows can be defined for completness and flexibility
155            my_skiprows = kwargs.get('skiprows') or 0
156            # ... separator can be defined, defalt is any whitespace
157            my_sep      = kwargs.get('sep') or r'\s+'
158            # ... usecols may be defined, default is the first two columns
159            my_usecols  = kwargs.get('usecols') or [0,1]
160            # ... columns may be re-defined OR default column names are used
161            my_columns  = kwargs.get('columns') or None
162            # (c) read the file with pd.read_csv and the **kwargs from above 
163            df = pd.read_csv(input_data, 
164                comment=my_comment, header=my_header, skiprows=my_skiprows, 
165                sep=my_sep, usecols=my_usecols)
166            if my_columns is not None: df.columns = my_columns
167            # (d) convert pd.DataFrame to np.ndarray
168            #     and transpose the array (columns should be the 1st index)
169            data = np.transpose(np.array(df))
170            # (e) set that the data are NOT the special ed.io.Profile
171            profile = None
172        elif isinstance(input_data, np.ndarray):
173            # The input is a numpy array.
174            # => just assign array to data
175            # => user's responsibility: ensure that the numpy.array
176            #    contains just two rows: X-values and Y-values
177            #    (for example, pass something like: arr[[0,2]]
178            name = 'np.ndarray'
179            data = input_data
180            profile = None
181        elif type(input_data) == pd.DataFrame:
182            # The input is pandas.DataFrame
183            # => convert DataFrame to numpy.array
184            # => user's responsibility: ensure that the pandas.Dataframe
185            #    contains just two columns: X-values and Y-values
186            #    (for example, pass something like: df[['Pixel','Iraw']]
187            # * we use type(input_data)==pd.DataFrame instead of isinstance
188            #   we need to do this due the the next option = {ediff.io.Profile}
189            #   => {ediff.io.Profile} is-instance of pd.DataFrame
190            #   => we would think that {Profiles} are DataFrames 
191            name = 'pd.DataFrame'
192            data = np.transpose(np.array(input_data))
193            profile = None
194        elif input_data.__class__.__name__ == 'Diffractogram1D':
195            # The input is ediff.io.Profile.
196            # (special case, input from our own, connected package ediff
197            # => convert Profile to numpy.array
198            # => the Profile has more-or-less fixed structure
199            #    the array is created from Profile columns {Pixel} a {Iraw}
200            # * we use var.__class__.__name__ instead of type
201            #   we DO NOT KNOW ediff.io.Profile here - it is not imported
202            #   we could import this, but then bground would depend on ediff
203            name = 'ediff.io.Diffractogram1D'
204            data = np.transpose(np.array(input_data[['Pixels','Iraw']]))
205            profile = input_data
206        else:
207            raise TypeError('Unknown type of input data!')
208         
209        # Return the result = 2xN numpy array with XY-data.
210        # (i.e. data[0] = X-data/values, data[1] = Y-data/values)
211        return(name, data, profile)

302        '''
303        Run the interactive plot.
304        
305        Technical notes
306        ---------------
307        * This method is very simple - it just runs the interactive plot.
308        * No parameters => everything is defined in InteractivePlot object.
309        '''
310
311        # Run interactive plot.
312        # No docstring: just class description above + comments below.
313        
314        # Brief description:
315        # * This method runs the interactive plot.

322        self.background.points.Y = []

334        
335        # Optimize the layout of the returned figure.
336        # (this works in all three supported interfaces: CLI, Spyder, Jupyter)
337        fig.tight_layout()
338        
339        # Show the plot
