Module: bground.api

Definition of API for (application programming interface) of BGROUND package.

  • The API can be employed as a simple UI within Spyder and/or Jupyter scripts.
  • The main purpose of the API - easy access to functions for bkgr subtraction.

Short example:

>>> # Semi-automated background subtraction with InteractivePlot method
>>> # (in Spyder, use %matplotlib qt before running this script
>>>
>>> # Import API of BGROUND package
>>> import bground.api as bkg
>>>
>>> # Define I/O files
>>> INPUT  = 'ed1_raw.txt'
>>> OUTPUT = 'ed2_bcorr.txt'
>>>
>>> # Define data, plot parameters and background subtraction method
>>> DATA = bkg.InputData(INPUT)
>>> PARS = bkg.OutputParams(OUTPUT,'Pixels','Intensity',xlim=200,ylim=180)
>>> BMET = bkg.InteractivePlot(DATA, PPAR, CLI=False)
>>>
>>> # Run the selected BMET = Background subtraction METhod 
>>> # (here: interactive plot = define bkgr manually + subtract automatically
>>> # (follow the instructions in stdout + newly opened window showing XY-data 
>>> BMET.run()

More examples for the individual bkgr subtraction methods:

  • Semi-automated/interactive bkgr subtraction: bground.api.InteractivePlot
  • Restore background from saved bkgr points: bground.api.RestoreFromPoints
  • Other, fully-automated methods - under development

113class InputData:
114    '''
115    Class defining {InputData} for {InteractivePlot}.
116    
117    * Input data can be: file, np.array, pd.DataFrame, ediff.io.Profile.
118    * The usage of InputData class is shown in the example above.
119    * The rest of the documentation => detailed comments in the source code.
120    '''
121    
122    def __init__(self, input_data, **kwargs):
123        # Initialization of InputData object.
124        # No docstring: just class description above + comments below.
125        
126        # Call read_input_data method with **kwargs:
127        # The method reads input data and returns:
128        #  - self.name = name of the data, filename or variable name
129        #  - self.data = numpy array with two rows ~ X-data, Y-data
130        # The input_data may be:
131        #  - text file containing several columns
132        #    (then **kwargs are passed to pd.read_csv
133        #  - numpy array
134        #    (user responsibility: array contains just two rows ~ X,Y
135        #  - pandas DataFrame
136        #    (user responsibility: DataFrame contains just two cols ~ X,Y)
137        #  - data saved in Profile object from ediff package
138        #    (program responsibility: df['pixel'] ~ X, df['Iraw'] ~ Y
139        self.name, self.data, self.profile = \
140            self.read_input_data(input_data, **kwargs)
141        
142        
143    def read_input_data(self, input_data, **kwargs):
144        # Read input data with XY-data.
145        # No docstring: just class description above + comments below.
146        
147        if isinstance(input_data, (str, Path)):
148            # The input is a file = filename given be str or PathLike var.
149            # => read the file using pd.read_csv
150            # => user's responsibility: pass suitable **kwargs
151            #    Sample kwargs: usecols=['Pix','Iraw'], comment='#'
152            #    Note: sep=r'\s+' is inserted as default, if not given.
153            name = 'File: ' + str(input_data)
154            # Set default separator to \s+
155            # (this is a convenience, as it is typical/default situation
156            if kwargs.get('sep') is None: kwargs['sep'] = r'\s+'
157            # Read the datafile to pd.DataFrame + convert it to np.array
158            data = pd.read_csv(input_data, **kwargs)
159            data = np.transpose(np.array(data))
160            profile = None
161        elif isinstance(input_data, np.ndarray):
162            # The input is a numpy array.
163            # => just assign array to data
164            # => user's responsibility: ensure that the numpy.array
165            #    contains just two rows: X-values and Y-values
166            #    (for example, pass something like: arr[[0,2]]
167            name = 'np.ndarray'
168            data = input_data
169            profile = None
170        elif type(input_data) == pd.DataFrame:
171            # The input is pandas.DataFrame
172            # => convert DataFrame to numpy.array
173            # => user's responsibility: ensure that the pandas.Dataframe
174            #    contains just two columns: X-values and Y-values
175            #    (for example, pass something like: df[['Pixel','Iraw']]
176            # * we use type(input_data)==pd.DataFrame instead of isinstance
177            #   we need to do this due the the next option = {ediff.io.Profile}
178            #   => {ediff.io.Profile} is-instance of pd.DataFrame
179            #   => we would think that {Profiles} are DataFrames 
180            name = 'pd.DataFrame'
181            data = np.transpose(np.array(input_data))
182            profile = None
183        elif input_data.__class__.__name__ == 'Profile':
184            # The input is ediff.io.Profile.
185            # (special case, input from our own, connected package ediff
186            # => convert Profile to numpy.array
187            # => array has more-or-less fixed structure
188            #    the array will contain Profile columns {Pixel} a {Iraw}
189            # * we use var.__class__.__name__ instead of type
190            #   we DO NOT KNOW ediff.io.Profile here - it is not imported
191            #   we could import this, but then bground would depend on ediff
192            name = 'ediff.io.Profile'
193            data = np.transpose(np.array(input_data[['Pixel','Iraw']]))
194            profile = input_data
195        else:
196            raise TypeError('Unknown type of input data!')
197         
198        # Return the result = 2xN numpy array with XY-data.
199        # (i.e. data[0] = X-data/values, data[1] = Y-data/values)
200        return(name, data, profile)

258        # Initialization of InteractivePlot object.
259        # No docstring: just class description above + comments below.
260        
261        # Brief description:
262        # * This object defines the interactive plot parameters.
263        # * The interactive plot is a simple user interface,
264        #   by means of which the background is defined and calculated.
265        # * InteractivePlot object takes three arguments/properties:
266        #   - DATA = InputData object above = input data
267        #   - PPAR = PlotParameters object above = plot params + output file
268        #   - CLI = set CLI=Trure if the program runs in command line interface

270        # Basic properties
271        # (DATA = InputData object
272        # (PPAR = PlotParameters object
273        # (these two objects should be defined BEFORE InteractivePlot object
274        self.data = DATA
275        self.pars = PARS

286        if CLI == True:
287            matplotlib.use('QtAgg')
288
289        # Messages property
290        # (If messages=True, short messages are printed on stdout
291        # (during the interactive plot processing after each keypress event.
292        # (The property is saved to
293        # (both InteractivePlot object and PlotParams object.
294        # (  => PlotParams.messages are transferred to InteractivePlot.run()
295        # (  => Therefore, the info about messages is readily available.
296        self.messages = messages
297        self.pars.messages = messages

300        '''
301        Run the interactive plot.
302        
303        Technical notes
304        ---------------
305        * This method is very simple - it just runs the interactive plot.
306        * No parameters => everything is defined in InteractivePlot object.
307        '''
308
309        # Run interactive plot.
310        # No docstring: just class description above + comments below.
311        
312        # Brief description:
313        # * This method runs the interactive plot.
314        # * In addition, it does a few minor things.
315        # * It takes no parameters - everything is in InteractivePlot object.

320        self.background.points.Y = []

332        
333        # Optimize the layout of the returned figure.
334        # (this works in all three supported interfaces: CLI, Spyder, Jupyter)
335        fig.tight_layout()
336        
337        # Show the plot
